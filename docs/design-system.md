# 设计系统说明

> 圆桌的界面方向不是科技冷光，而是“纸上开席、灯下论道”的古诗风。

## 气质

- 画面以暖纸色、墨色、朱砂色为主
- 信息密度可以高，但节奏要从容，不做强刺激仪表盘
- 用户看到的应该是“论辩、收束、展卷”，而不是技术控制台

## 颜色与材质

- 主背景：温润纸面色，避免纯白平铺
- 主文字：高对比墨色
- 强调色：朱砂 / 金色系，用于 CTA、焦点与状态强调
- 容器层次：靠轻边框、暖阴影与半透明卡片区分，不靠重黑线

## 字体与层级

- 页面主气质用衬线字承担
- 功能型按钮、表单、标签可用 sans
- 标题要像卷首题签，正文要像批注，不要出现默认 SaaS 风的排版

## 组件原则

- 交互组件要给清晰反馈，但不过度跳动
- 卡片、弹层、时间线都应服务于“席间叙事”，而不是堆砌 UI 花样
- Markdown 展示优先保证可读性，长文要有节奏感和留白

## 动效原则

- 只保留有叙事价值的过渡，如入场、显影、状态切换
- 动效要轻、短、稳，不制造界面焦虑

## 页面语义

- 首页：序页 / 入席引导
- 圆桌页：设题、选席、观辩、插话、结案
- 法庭页：同一编排能力的另一种舞台化表达
- 分享页 / 旧席页：展卷、续席、复刻

## 落地要求

- 新 UI 改动优先复用现有 token 与版式语言
- 不引入突兀的冷色科技风插图或默认模板图标
- 若新增大型页面，先补一页对应的设计说明再扩展实现
